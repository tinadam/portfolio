US Department of Education, a case study


The US Department of Education website was redesigned for improved navigation, simpler display of information, and a more welcoming feel



Timeline: 4 weeks
Project Context: Partner and solo work for UX/UI bootcamp
My Roles: UX research, UX design, UI design
Primary Tools: Figma, InVision

Jump to:  Researching  /  Ideating  /  Prototyping and Testing  /  Iterating  /  Reflecting










Researching


current website analysis
We evaluated the current website based on heuristics, accessibility, and usability.

Key areas for improvement included:
  • Ensuring that content is not repetitive and that important information is displayed prominently

  • Reorganizing the navigational structure and adding more specificity to link names

  • Updating the site design so that it continues to sport a minimalistic look but also appear clean and modern





user research plan
My teammate and I hypothesized that users would mostly be students, teachers, and administrators. For this project, we focused our proto-persona and user persona on the student.

We thought that students would most likely visit the website for information regarding loans and grants, so we asked users to navigate those pages on the current site to find out how easy the experience was for them.






key takeaways




We found that:
  • 2/3 of users opted for the Search function

  • 5/6 of users mentioned that there was too much text to scan through

  • 2/3 of users pointed out that wording related to the navigation was confusing

  • 1/2 of users did not know how to return to the home page

  • 1/2 of users verbally mentioned that website was not friendly






user persona






summary
The current Department of Education website may be confusing and overwhelming for student users. Based on user interviews, the current site may benefit from updates that make the site more approachable and reduce information overload.

Ideating


feature matrix
Using the same set of research, my teammate and I proceeded individually from this point onward.

I decided to focus on simplifying primary and secondary navigation, improving wording, redesigning the search page, and updating the visual design for a more modern and friendly feel.






card sorting
Laying out the links on sticky notes helped me group content together as well as identify opportunities to redistribute or delete links.






site mapping
I ordered the primary and secondary content by category, hierarchy of importance based on user testing, and alphabetically otherwise.






summary
I wanted to refresh the website by updating navigational structure, the search option, copy specificity, and visual design.

To simplify navigation and eliminate confusion, I consolidated content when possible, deleted repeated links, added more specificity to link names, deleted the sidebar navigation, and redistributed those pages into appropriate secondary pages.

Prototyping and Testing

mobile wireframes
  
I started with mobile screens to ensure responsiveness.






desktop wireframes
I condensed the header and footer. Since the Search option was popular among users, I brought the "About" page to the footer section in order to create room for it.






5 second usability testing
I grabbed a handful of users to get their quick gut impressions. One user noted that she liked how prominent Accessibility options were and a different user noted that they liked the one-column layout.

Based on user reactions, I moved “Other Resources” down to the footer navigation area since it seemed of lesser importance. Under "Laws and Guidance," I created a “Legislation” secondary page, rather than listing all Acts out as secondary links.







summary
I designed mobile wireframes, then desktop wireframes, and then tested some users on the navigation. I continued making adjustments so that the main navigation only housed necessary content and more attention was placed on the Accessibility and Search features.

Iterating


brand style


The new style guide retains aspects of original branding but adopts a UI that hopefully creates a supportive, informative, and clean environment for users.






higher fidelity prototype
     






further user testing
At this point, I executed another round of user testing with three tasks to see if users can locate content and complete a navigational path.

Based on these results, I:
  • Enlarged the hamburger menu and moved it to the right side beneath other links to increase prominence

  • Changed the “Legislation” secondary page name to "Legislation (Acts and Laws)" for more clarity

  • Moved About page back into main navigation bar—although it may not be as frequently used, many users seemed to expect it as a primary link

  • Cleaned up Search page and moved the search bar up based on user preferences

Reflecting


future direction
Given the time and scope, I would've loved to spend more time with the following:
  • Expanding accessibility options for equitable browsing experiences

  • Considering different navigational flows for different user types (how might the flow differ for teachers or administrators, if at all?)

  • Revisiting the copy and tone to ensure reachability with all potential users





final thoughts
This project was challenging because I wanted to give the Department of Education website a refresh while respecting its original branding and existing system. It was fun to try and balance while making sure that I never lost sight of the main goals for this project.

I also enjoyed seeing what my teammate came up with, since we shared user research but designed different sites. There are so many ways to go about this, and seeing her work was another good reminder that there's always another angle to explore.
